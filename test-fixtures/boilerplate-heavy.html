API Reference Guide

This comprehensive guide covers all available API endpoints. Our API uses RESTful conventions and returns JSON responses.

Authentication

All API requests require authentication using Bearer tokens. Include your API key in the Authorization header.

Tokens expire after 24 hours and must be refreshed using the refresh endpoint.

Authorization: Bearer your-api-key-here

Rate Limiting

The API implements rate limiting to ensure fair usage. Free tier users are limited to 100 requests per hour.

When you exceed your rate limit, you will receive a 429 status code. The response headers include X-RateLimit-Reset.

Error Handling

All errors return a consistent JSON structure with an error code and message. Client errors use 4xx status codes.
