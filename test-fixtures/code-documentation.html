TypeScript Configuration Reference

The tsconfig.json file specifies the root files and compiler options required to compile a TypeScript project.

Compiler Options

The compilerOptions property contains settings that control how TypeScript compiles your code.

Strict Mode Options

Enabling strict mode activates a family of strict type-checking options. This is recommended for all new projects.

{
  "compilerOptions": {
    "strict": true,
    "noImplicitAny": true,
    "strictNullChecks": true,
    "strictFunctionTypes": true,
    "noUncheckedIndexedAccess": true
  }
}

Module Resolution

TypeScript supports several module resolution strategies. For modern projects, use "NodeNext" or "Bundler".

"moduleResolution": "NodeNext",
"module": "NodeNext",
"target": "ESNext"

Include and Exclude

The include array specifies which files to compile. Use glob patterns to match files.

The exclude array specifies which files to skip. By default, node_modules is excluded.

{
  "include": ["src/**/*.ts"],
  "exclude": ["node_modules", "dist", "**/*.test.ts"]
}

Project References

Project references allow you to structure TypeScript programs into smaller pieces. Each sub-project has its own tsconfig.json.

This improves build times through incremental compilation and enables better code organization in monorepos.
